# X-Mars 对比 Claude Code / Hermes 工具能力实施方案

> 阶段：TCH（Tooling Claude Hermes）
> 设计来源：用户请求“https://github.com/nousresearch/hermes-agent 对比 x-mars 优劣，仔细分析哪些工具模块可以吸取”及后续“加上之前的 Claude Code 对比”

> 更新时间：2026-05-03

## 阶段摘要

本阶段把 X-Mars、Claude Code、Hermes Agent 的工具能力对比转为可执行实施账本。实施策略不是复刻 Claude Code 的闭源终端产品，也不是把 Hermes Python 单体能力整体移植，而是在 X-Mars 现有 TypeScript 包边界内吸收成熟能力：

- 从 Claude Code 吸收：`WebFetch` / `WebSearch` 工具语义、Skill 文件化加载模型、配置层级、权限 DSL、命令式 hooks、文件化 subagent、CLI/CI 可组合性。
- 从 Hermes Agent 吸收：多 provider web tools、大输出落盘、FTS 会话搜索、programmatic tool calling、cron/gateway、skill readiness。
- 保留 X-Mars 差异化：`@x-mars/hooks` 权限链、`@x-mars/agent` deferred tools 与并发分区、`@x-mars/swarm` 协作抽象。

## 工具逻辑实现对比

### web_fetch

| 项目        | 当前实现                                                                                                                           | 优势                                                       | 缺口                                                                               | 吸收方向                                                                                                     |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| X-Mars      | `web_fetch` 直接 `fetch()`，校验 URL，5MB 响应上限，HTML 转 text/markdown/raw，JSON pretty print，按 `TOOLS_MAX_OUTPUT_BYTES` 截断 | 简单、类型安全、无 key 可用、和 `ToolResult`/Hook 管线一致 | 不能处理 JS-heavy 页面；无 Claude Code 式权限/工具语义契约；截断后无法恢复完整内容 | 保留 fast path，schema、权限、错误呈现和工具说明参考 Claude Code `WebFetch`；provider/extractor 作为底层扩展 |
| Claude Code | 内置 `WebFetch`，面向 URL 内容读取，配合权限、settings、MCP 扩展和 `WebSearch` 使用路径                                            | 产品体验稳定，权限模型成熟，工具职责清晰                   | 内部实现不可复用                                                                   | 作为主参考：对齐工具语义、权限 gating、URL/domain 约束、错误输出和与 search 的协作说明                       |
| Hermes      | `web_extract_tool` 支持 Exa/Firecrawl/Parallel/Tavily，配置 backend，可通过辅助模型提取摘要                                        | 生产弹性强，适合复杂网页和研究任务                         | 依赖外部 key/provider；Python 实现与 X-Mars 架构不兼容                             | 仅作为 provider adapter 和 LLM extract 压缩的次级参考                                                        |

### web_search

| 项目        | 当前实现                                                                          | 优势                                 | 缺口                                                                                                 | 吸收方向                                                                                        |
| ----------- | --------------------------------------------------------------------------------- | ------------------------------------ | ---------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| X-Mars      | 抓取 Brave Search HTML，用正则解析 title/url/snippet                              | 无 API key，默认可用                 | HTML 结构脆弱；无 Claude Code 式 domain filter、权限 gating、recency/source 参数和稳定 result schema | 以 Claude Code `WebSearch` 为产品契约，抽象 provider，Brave 只作为 fallback                     |
| Claude Code | 内置 `WebSearch`，支持搜索、权限控制、domain filtering，并自然引导后续 `WebFetch` | 用户体验稳定，和 `WebFetch` 配合自然 | 不暴露底层实现                                                                                       | 作为主参考：补 domain filtering、权限 gating、工具说明、结果格式和错误行为                      |
| Hermes      | 多 provider search，和 extract/crawl 共用 backend 配置                            | 可靠性和可配置性更强                 | provider 成本和配置复杂度更高                                                                        | 仅作为 provider adapter 的次级参考，可选接 Firecrawl/Tavily/Exa/Parallel 或 MCP search provider |

### skill

| 项目        | 当前实现                                                                                                             | 优势                                                      | 缺口                                                                                                       | 吸收方向                                                                                                                    |
| ----------- | -------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| X-Mars      | `skill_search` 由宿主注入 search，`skill_load` 读取指定 `SKILL.md`；已有 create/execute/improve 工具文件             | 类型边界干净，适合接入 registry/provider                  | 缺 Claude Code 式用户/项目/bundled/MCP 加载层级、catalog 注入、invoked skills 记录和 linked files 统一视图 | 以 Claude Code SkillTool / loader 模型为主，补 `.x-mars/skills`、bundled skills、MCP skills、catalog section 和按需正文加载 |
| Claude Code | Skills 采用用户级、项目级、bundled、MCP 来源；SkillTool 按需调用，catalog/commands 进入上下文，正文不全量污染 prompt | 人类可维护，团队协作清晰，产品语义成熟                    | 闭源实现不可复用                                                                                           | 作为主参考：对齐加载层级、frontmatter、catalog 注入、按需加载、invoked skills 记录                                          |
| Hermes      | `skills_list` 只列 metadata，`skill_view` 按需加载正文或 linked files；支持 platform/prerequisites/setup             | progressive disclosure 和 readiness 完整，适合大 skill 库 | 功能密集且和 Hermes home/config 绑定                                                                       | 仅吸收 readiness/setup/linked files/usage bump，作为 Claude Code skill 模型的增强字段                                       |
